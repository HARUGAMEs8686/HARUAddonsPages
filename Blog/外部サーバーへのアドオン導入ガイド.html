BDS(外部サーバー) 導入・アップデートガイド

BDSへのアドオン新規導入手順

Bedrock Dedicated Server (BDS) などの外部サーバーにアドオンを導入する手順です。

1. アドオンファイルの解凍

ダウンロードしたアドオン（.mcaddon または .zip）を解凍し、中にあるビヘイビアーパック(BP)とリソースパック(RP)のフォルダを取り出します。

2. サーバーのフォルダへ配置

取り出したフォルダを、サーバーのディレクトリ内にある指定のフォルダへアップロードします。

  • BPフォルダ ➔ behavior_packs または development_behavior_packs
  • RPフォルダ ➔ resource_packs または development_resource_packs

3. ワールドへの適用 (JSONファイルの編集)

サーバー内の worlds/Bedrock level/ (※ワールド名によって異なります) の中にある、以下の2つのファイルにアドオンのUUIDとバージョンを追記します。ファイルがない場合は新規作成してください。

▼ world_behavior_packs.json の記述例

[
  {
    "pack_id": "BPのUUIDを入力",
    "version":[1, 0, 0]
  }
]

▼ world_resource_packs.json の記述例

[
  {
    "pack_id": "RPのUUIDを入力",
    "version":[1, 0, 0]
  }
]
注意: 複数のアドオンを入れる場合は , (カンマ) で区切って配列の中に追加してください。最後の要素にカンマをつけるとエラーで読み込まれません。

アドオンのアップデート手順

既に導入しているアドオンを最新バージョンに更新する場合の手順です。

  1. サーバーを停止する (起動したまま書き換えるとワールドが破損する恐れがあります)
  2. behavior_packs および resource_packs 内の古いアドオンフォルダを完全に削除する
  3. 新しいバージョンのBP/RPフォルダをそれぞれ配置する
  4. world_behavior_packs.json と world_resource_packs.json に記載しているバージョン(version)の数値を最新のものに書き換える
  5. サーバーを起動する
