ServerTransfer

戻る

アドオンの概要

ServerTransferは、UIやコマンドから簡単に別サーバーへ移動できるアドオンです。Bedrock Dedicated Server (BDS) などで複数のワールドを繋ぐハブサーバーを構築する際に役立ちます。

アドオンの入手

ServerTransferは以下のボタンからダウンロードできます。

【mcaddon】ServerTransfer_v1.3.0 【zip】ServerTransfer_v1.3.0

・v1.3.0: コマンドのみで、サーバー接続が可能に

・v1.2.0: UUIDの修正(別アドオンと干渉する不具合を修正しました)

・v1.1.0: セレクターの不具合修正&アップデート時にアドオンの故障のリスクを軽減

機能と設定

このアドオンは、簡単なコマンドでサーバー移動UIを表示したり、直接プレイヤーを転送させることが可能です。設定ファイル `scripts/config.js` を編集することで、UIに表示される移動先サーバーを自由にカスタマイズできます。

設定方法 (UI用)

UIに表示するサーバーリストは、アドオン内の `scripts/config.js` ファイルを編集して設定します。`name` にはUIに表示する名前、`ip` と `port` には接続先サーバーの情報を入力してください。

// scripts/config.js の例
export const config = [
    {
        name: 'ロビーサーバー',
        ip: '000.000.000.000',
        port: 19132,
    },
    {
        name: 'ゲームサーバー',
        ip: '111.111.111.111',
        port: 19133,
    },
];

使用方法

サーバー移動には「UIを表示する方法」と「コマンドで直接転送する方法」の2種類があります。

1. UIを表示して移動

以下のコマンドを実行すると、対象のプレイヤーに `config.js` で設定したサーバーリストUIを表示できます。

/scriptevent servertransfer:ui <ターゲットセレクター>

実行例:
・自分にUIを表示: /scriptevent servertransfer:ui @s
・全てのプレイヤーに表示: /scriptevent servertransfer:ui @a

2. コマンドで直接サーバーへ転送する

以下のコマンドで、UIを介さずに直接プレイヤーを特定のサーバーに転送できます。

/scriptevent servertransfer:go <サーバーセレクト> <セレクター>
  • <サーバーセレクト>: config.jsで設定したサーバーの番号（0から始まります）。
  • <セレクター>: 転送したいプレイヤーを指定します（例: @a, @s, プレイヤー名）。

実行例:

/scriptevent servertransfer:go 0 @a

この例では、全てのプレイヤー（@a）をconfig.jsの0番目（最初）に設定されたサーバーへ転送します。

導入方法

以下の手順でアドオンを導入してください：

  • ・ワールド設定でチートを有効にする
  • ・「ベータAPI」を有効にする
注意: このアドオンはプレイヤーを別サーバーに転送する機能を使用します。そのため、Bedrock Dedicated Server (BDS) など、サーバー転送が許可されている環境でのみ動作します。通常のローカルワールドでは機能しません。
